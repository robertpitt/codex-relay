# Simplify create ticket modal to agent-generated drafts only

## Context

The create ticket modal currently exposes manual entry areas for title, labels, and markdown, plus a Manual Draft option. These fields should no longer be shown because the agent draft process owns ticket title, labels, and description generation. After the agent creates a draft, the modal header should reflect the draft content: the dialog title should become the draft title, and the subtext should become the draft summary, falling back to the first roughly 150 characters of the generated draft description/markdown.

## Research Findings

- src/renderer/src/App.tsx is the primary renderer surface found for ticket drafting UI; matches include TicketDraftErrorPayload and MarkdownBlock, suggesting the create ticket modal/draft preview logic is likely centralized there.
- tests/ticket-draft.test.ts covers the draft pipeline helpers createTicketDraft, draftToCreateInput, and extractTicketDraftUrls; update or extend these tests if draft-to-create behavior changes.
- src/main/index.ts contains IPC-related matches for createTicketDraft, TicketDraftResult, TicketCreateInput, and ticket:createDraft, so renderer changes should preserve the existing draft IPC contract.
- src/main/services/codex.ts defines CreateDraftInput and related Codex run types, indicating the agent draft process is already modeled in the backend service layer.
- src/shared/types.ts includes shared ticket/project types and settings such as ticketDraftingEnabled; confirm whether the Manual Draft option is only UI state or also represented in shared types/settings before removing it.
- tests/backend.test.ts covers createTicket and related backend flows; use existing tests as a guardrail if removing manual draft UI changes ticket creation assumptions.

## Requirements

- Remove the Manual Draft option from the create ticket modal UI.
- Do not show manual input areas for title, labels, or markdown in the create ticket modal.
- Keep the rough idea input/workflow available so users can start an agent draft.
- Ensure ticket title, labels, and markdown/description are populated from the agent-created draft rather than manual modal fields.
- After an agent draft is created, change the dialog title/header to the generated draft title.
- After an agent draft is created, change the dialog description/subtext to the draft summary when available, otherwise a clean first ~150 characters from the draft body/markdown.
- Preserve existing ticket creation behavior after accepting/creating from an agent draft.
- Remove dead state, handlers, validation, and tests tied exclusively to the Manual Draft option if they are no longer used.

## Implementation Plan

- Inspect src/renderer/src/App.tsx to identify the create ticket modal component/state and the UI branch that renders Manual Draft versus agent draft controls.
- Remove the Manual Draft selector/option and any conditional rendering path that displays title, labels, or markdown text inputs inside the modal.
- Keep only the rough idea prompt/input and agent draft action path before a draft exists; ensure validation no longer expects manual title/markdown fields from the modal.
- Update post-draft modal state so the dialog heading uses draft.title once TicketDraftResult is available.
- Add a small renderer helper for dialog subtext: prefer draft.summary if the schema exposes one; otherwise derive text by stripping simple markdown from the generated markdown/body and truncating to about 150 characters without cutting awkwardly when practical.
- Confirm the create action still uses the existing draftToCreateInput-style data path or equivalent agent draft payload, including labels generated by the draft.
- Remove unused imports, state variables, and handlers related to manual title/labels/markdown entry and Manual Draft mode.
- Update tests in tests/ticket-draft.test.ts and any renderer tests/snapshots if present to reflect an agent-draft-only modal and the new post-draft title/subtext behavior.
- Run the relevant test suite for ticket draft creation and, if the project has renderer tests, run the focused frontend tests for App/create ticket modal behavior.

## Acceptance Criteria

- The create ticket modal no longer displays a Manual Draft option.
- The create ticket modal no longer displays editable title, labels, or markdown fields.
- A user can still enter a rough idea and trigger the agent draft process.
- Once the draft is created, the modal header/title displays the generated draft title.
- Once the draft is created, the modal subtext displays the draft summary or a clean ~150-character fallback from the draft content.
- Creating a ticket from the draft still sends generated title, labels, and markdown/description to the existing create-ticket flow.
- No unused Manual Draft state, handlers, or imports remain in the renderer.
- Relevant tests pass, including existing ticket draft tests.

- Research was limited to the listed files and did not include a full component tree trace, so the modal may be split into helper components imported by src/renderer/src/App.tsx.
