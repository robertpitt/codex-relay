# Update primary color for dark theme

## Context

The app's current primary color should be changed to a darker-theme-compatible color. The goal is to make primary UI elements feel more appropriate in the existing dark theme while preserving readability, accessibility, and visual consistency across the product.

## Requirements

- Locate the app's primary color definition, preferring existing theme tokens, CSS variables, Tailwind config, design system constants, or shared style files over one-off component edits.
- Replace the current primary color with an approved dark-theme-suitable value and update related states such as hover, active, focus ring, disabled, border, and background variants if they are derived separately.
- Apply the change consistently to primary buttons, links, selected states, navigation highlights, form focus states, badges, and any other UI that consumes the primary color token.
- Preserve accessible contrast for text, icons, focus indicators, and interactive states in dark mode.
- Avoid unrelated layout, copy, or component refactors.

## Acceptance Criteria

- Primary-colored UI elements use the new dark-theme-compatible primary color consistently across the app.
- Hover, active, selected, disabled, and focus states still look intentional and remain distinguishable.
- Text and icon contrast meets WCAG AA where applicable, especially on primary buttons and highlighted surfaces.
- No obvious hardcoded old-primary color remains in visible primary UI surfaces unless intentionally documented.
- Existing tests, linting, or style checks pass, or any failures are documented as unrelated.

## Clarification Questions

- What exact primary color should be used, or should the implementer propose 2-3 options for approval?
- Should this change affect only dark mode, or should it also update the app's default/light theme primary color?
- Are there brand guidelines or an existing palette that the new primary color must follow?

## Implementation Notes

- Start by searching for theme-related files such as Tailwind config, CSS variables, design tokens, global styles, or shared UI component theme definitions.
- Prefer changing the central primary color token so existing components inherit the update naturally.
- If the app has generated color scales, update the source color and regenerate or adjust the scale using the existing project convention.
- Take before/after screenshots of representative screens with primary buttons, navigation selection, forms, and links to verify consistency.
